# Feature Specification: iOS 原生體驗與明確貼上

**Feature Branch**: `main`

**Created**: 2026-07-17

**Status**: active
**Owner**: NextPaste

**Input**: 使用者要求重新設計目前未完整符合 iOS 慣例的畫面，保留既有設計語言，並讓使用者在其他 App 複製內容後打開 NextPaste 即可使用；整體必須符合 Apple 官方設計與隱私指南。

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 複製後開啟並明確貼上目前內容 (Priority: P1)

使用者在其他 App 複製一段文字或圖片後開啟或返回 NextPaste，畫面立即提供真實、可見的系統 `PasteButton`。只有使用者點擊該控制後，系統才把目前相容內容交給 NextPaste 驗證與保存；App 不在啟動、畫面出現或前景生命週期中自動讀取其他 App 的剪貼簿內容。

**Why this priority**: 剪貼簿是產品的主要資料來源；目前 iOS 完全無法取得剪貼簿內容，導致核心價值不可用。

**Independent Test**: 在另一個 App 複製一筆有效文字或圖片，接著啟動或切回 NextPaste；先確認未點貼上前沒有新增項目或程式化貼上權限提示，再點擊系統 `PasteButton`，確認內容只新增一次且立即出現在歷史中。

**Acceptance Scenarios**:

1. **Given** 使用者在其他 App 複製一筆有效文字，**When** 使用者開啟或切回 NextPaste，**Then** 畫面必須立即提供系統 `PasteButton`，且未點擊前不得讀取或保存該文字。
2. **Given** 使用者點擊可見的系統 `PasteButton`，**When** 系統交付一筆有效文字或受支援圖片，**Then** 該內容必須經既有驗證、去重與保存流程後出現在歷史中。
3. **Given** 目前剪貼簿內容已存在於歷史，**When** 使用者再次點擊系統 `PasteButton`，**Then** 歷史不得新增重複項目，並提供不含內容的回饋。
4. **Given** 使用者只開啟或切回 NextPaste 而未點擊 Paste，**When** scene 進入 active，**Then** App 只能刷新自有資料，不得呼叫跨 App pasteboard value API 或顯示由程式化讀取造成的貼上權限提示。
5. **Given** NextPaste 位於背景或未執行，**When** 剪貼簿內容改變，**Then** App 不得宣稱已在背景保存；使用者返回後只能明確貼上當下仍存在的最近內容。

---

### User Story 2 - 在 iPhone 上使用原生且完整的主畫面 (Priority: P1)

使用者在任何支援的 iPhone 尺寸上都能看到完整的標題、搜尋、歷史列表與主要操作，不會出現水平裁切、桌面式工具列或超出安全區域的控制項。

**Why this priority**: 目前畫面使用桌面最小寬度與桌面工具列，導致 iPhone 上的重要內容及操作被裁切。

**Independent Test**: 在最窄與最大支援 iPhone 尺寸、直向與橫向、標準及較大文字尺寸下開啟首頁，確認所有主要內容可見、可捲動且可操作。

**Acceptance Scenarios**:

1. **Given** 使用者以直向 iPhone 開啟 App，**When** 首頁完成顯示，**Then** 標題、搜尋、篩選、新增、設定與歷史內容必須完整位於可見區域內。
2. **Given** 使用者使用大文字或輔助使用者介面尺寸，**When** 文字與控制項放大，**Then** 內容必須自然換行或捲動，不得截斷必要文字或產生水平畫布。
3. **Given** 使用者搜尋歷史，**When** 輸入關鍵字，**Then** 結果必須隨輸入更新，且畫面只提供一個明確的搜尋入口。
4. **Given** 使用者選擇歷史篩選條件，**When** 篩選套用，**Then** 畫面必須顯示目前條件並提供容易找到的還原方式。
5. **Given** 使用者使用 VoiceOver、鍵盤或切換控制，**When** 導覽首頁，**Then** 所有主要動作必須有清楚名稱、順序、狀態與可達操作。

---

### User Story 3 - 以 iOS 慣例新增與管理剪貼項目 (Priority: P2)

使用者可以透過原生導覽列新增文字項目、搜尋與篩選歷史，並以清楚的點按、滑動或選單操作複製、釘選及刪除項目。

**Why this priority**: 核心資料已能保存，但目前互動把多個桌面控制擠在同一列，且部分觸控目標不足。

**Independent Test**: 在 iPhone 上建立一筆手動文字項目，接著複製、釘選、取消釘選及刪除，確認每個動作都有清楚回饋且資料狀態正確。

**Acceptance Scenarios**:

1. **Given** 使用者點選新增，**When** 編輯畫面出現，**Then** 取消與儲存必須位於符合 iOS 慣例的位置，並能正確處理鍵盤與驗證訊息。
2. **Given** 歷史中有一筆項目，**When** 使用者執行複製，**Then** 系統必須更新剪貼簿並提供不遮擋內容的成功回饋。
3. **Given** 歷史中有一筆項目，**When** 使用者透過原生滑動或選單執行釘選或刪除，**Then** 項目狀態必須正確更新，且刪除動作必須明確呈現破壞性角色。
4. **Given** 使用者以觸控操作任何圖示控制，**When** 點按控制，**Then** 可點按區域不得小於 iOS 建議的最小觸控尺寸。

---

### User Story 4 - 在 iOS 上使用真正的設定介面 (Priority: P2)

使用者可以從首頁進入符合 iOS 慣例的設定畫面，調整語言、外觀與歷史上限，查看本機與剪貼簿隱私說明，並在確認後清除歷史；macOS 專屬快捷鍵不會出現在 iOS。

**Why this priority**: 目前 iOS 的設定按鈕只顯示「尚未提供」訊息，使既有偏好與資料管理無法使用。

**Independent Test**: 從 iPhone 首頁開啟設定，逐一調整語言、外觀與歷史上限，再重新開啟 App，確認設定保存；執行清除動作時確認必須先經過清楚的破壞性確認。

**Acceptance Scenarios**:

1. **Given** 使用者位於首頁，**When** 開啟設定，**Then** 必須顯示可導覽、可捲動且不超出螢幕的原生設定介面。
2. **Given** 使用者變更語言或外觀，**When** 返回首頁，**Then** 畫面必須立即使用新偏好，且下次啟動仍保留。
3. **Given** 使用者調整歷史上限，**When** 提交有效值，**Then** 系統必須保存正規化後的值並遵守既有保留規則。
4. **Given** 使用者選擇清除未釘選或全部歷史，**When** 確認對話顯示，**Then** 對話必須說明影響數量與不可復原性，且取消不得改變資料。

### Edge Cases

- App 首次啟動時剪貼簿已包含在其他 App 複製的敏感內容；未點 Paste 前不得接觸其 value。
- 系統或 App-specific 設定對程式化貼上存取有限制；基本流程仍不得依賴該設定。
- App 在背景期間剪貼簿被連續覆寫多次；回到前景時只能取得最後仍存在的一筆。
- 剪貼簿為空、只有空白文字、不支援的資料類型、損壞圖片或超過既有大小限制的圖片。
- 同一份內容被使用者連續點擊 Paste，或兩次明確貼上請求快速重疊。
- 使用者在明確貼上處理尚未完成時迅速切到背景再回到前景。
- 搜尋或篩選結果為空時，畫面需要提供與空歷史不同的復原動作。
- iPhone 橫向、分割畫面、顯示縮放、最大輔助文字尺寸及 VoiceOver 開啟時的版面。
- 列表內容包含長文字、由右至左文字、圖片、釘選與未釘選項目的混合。
- 使用者在設定或新增項目畫面顯示期間切換前景與背景。

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: iOS App 在啟動、`onAppear`、`.task`、notification、timer 或 `scenePhase` 進入 active 時不得自動存取 `UIPasteboard.general` 的 item value 或 providers；生命週期回呼只能刷新 App 自有資料與畫面狀態。
- **FR-002**: 只有使用者實際點擊真實且可見的 SwiftUI `PasteButton`（或等價系統 paste control）後，系統交付的 payload 才能進入既有偵測、驗證、去重、保存與畫面更新流程。
- **FR-003**: 系統必須支援從 iOS 剪貼簿匯入既有產品可接受的純文字與點陣圖片類型。
- **FR-004**: 貼上控制必須是真實、可見、可點擊、對比足夠的系統控制，且不得以普通按鈕、透明控制、覆蓋控制、程式模擬點擊或其他方式繞過其使用者意圖與隱私語意。
- **FR-005**: 空歷史必須在空狀態中提供系統 `PasteButton` 作為唯一明顯的主要動作；已有歷史時必須在 trailing toolbar 提供該主要動作，且同一畫面不得重複呈現競爭的 Paste primary action。
- **FR-006**: 系統不得在 iOS 背景、App 未執行或單純回到前景時監測或保存剪貼簿；只在明確貼上時處理當下仍存在的相容內容。
- **FR-007**: iOS 剪貼簿處理必須保持完全在裝置上，且狀態、錯誤與診斷不得暴露實際文字、圖片或預覽內容。
- **FR-008**: 系統必須序列化或以請求所有權保護同一 App 生命週期中的明確貼上請求，使取消、被較新請求取代或晚到的非同步結果不得重複保存內容。
- **FR-009**: iOS 首頁必須使用平台原生的導覽層級、標題、工具列與搜尋呈現，不得重用需要桌面寬度的卡片式頂端工具列。
- **FR-010**: iOS 首頁必須在所有支援 iPhone 尺寸上避免水平裁切，並尊重安全區域、直向、橫向與系統顯示縮放。
- **FR-011**: iOS 首頁必須只提供一個明確搜尋入口，搜尋輸入時必須即時更新結果並可清除。
- **FR-012**: iOS 首頁必須提供新增、篩選、設定及歷史管理入口；次要與破壞性動作可以收納於原生選單，但不得隱藏主要新增或貼上流程。
- **FR-013**: iOS 歷史列表必須以每筆資料的穩定識別維持內容與操作狀態，插入、刪除、搜尋、篩選及釘選重排不得錯配其他列。
- **FR-014**: iOS 列表必須提供原生且一致的複製、釘選、取消釘選與刪除操作，並對成功、重複、無內容、不支援與失敗結果提供不含敏感內容的回饋。
- **FR-015**: 所有 iOS 互動控制必須提供至少 44 x 44 點的有效觸控區域，並具有可本地化的名稱、提示、狀態及 VoiceOver 操作。
- **FR-016**: iOS 版面與文字必須支援動態字級、粗體文字、增加對比、減少透明度及減少動態效果，不得以裁切必要資訊作為適配方式。
- **FR-017**: 首次空歷史必須說明「在其他 App 複製後返回」的產品流程，並提供貼上與手動新增入口；搜尋及篩選空結果必須提供對應的清除搜尋或重設篩選動作。
- **FR-018**: iOS 手動新增畫面必須使用符合平台慣例的導覽、取消與儲存操作，支援鍵盤、輸入驗證與錯誤恢復，且不得要求桌面最小寬度。
- **FR-019**: iOS 必須提供真正可用的設定介面，至少包含語言、外觀、歷史上限、資料與剪貼簿隱私說明、清除未釘選歷史、清除全部歷史及版本資訊。
- **FR-020**: iOS 設定不得顯示只適用於 macOS 的全域快捷鍵或 Mac 專屬文案；macOS 現有設定與主畫面行為必須保持不變。
- **FR-021**: 現有暖米色畫布、金色與其他既有點綴色、圓角卡片、字體層級、徽章、插圖及成功／錯誤語意角色必須沿用共享設計系統，不得另建平行品牌樣式。
- **FR-022**: 本功能不得改變既有本機資料來源、去重資格、圖片持久化、歷史排序、保留上限或離線可用性。
- **FR-023**: iOS 明確貼上、啟動／前景無程式化讀取、首頁版面、設定與新增流程必須有可重複的自動驗證；系統 paste control 與平台設定中無法可靠自動化的部分必須有明確人工驗證紀錄。
- **FR-024**: iOS 專屬行為不得意外改變 macOS、visionOS 或其他未列入本功能的 Apple 平台行為與支援矩陣。

### Key Entities

- **Explicit Paste Request**: 使用者點擊系統 `PasteButton` 後產生的一次本機非同步請求，包含請求所有權與處理狀態，但不保存內容預覽。
- **Clipboard Import Result**: 一次使用者主動貼上的結果，區分已保存、重複、空白、不支援、取消及失敗，不包含實際敏感內容。
- **iOS Navigation State**: 首頁、設定與新增畫面的呈現狀態，以及搜尋文字與歷史篩選條件。
- **User Preference**: 既有語言、外觀與歷史上限設定，持續使用原本的本機保存規則。
- **Captured Item**: 既有文字或圖片剪貼項目；本功能只新增 iOS 取得來源，不變更資料模型或保存資格。

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 在另一個 App 複製有效文字後冷啟動或返回 NextPaste 的 20 次重複驗證中，點擊可見系統 `PasteButton` 後內容於 1 秒內出現在歷史的成功率為 100%，且每次只新增一筆。
- **SC-002**: 在受支援圖片的 10 次明確貼上驗證中，圖片保存、縮圖顯示及重新開啟後還原成功率為 100%，且無重複項目。
- **SC-003**: 在 20 次 copy → open／foreground 驗證中，點擊 Paste 前由程式化讀取造成的權限提示與內容新增次數皆為 0；Paste 控制在空歷史立即可見，已有歷史時可在一次點按內使用，敏感內容洩漏與閃退次數皆為 0。
- **SC-004**: 在最窄與最大支援 iPhone、直向與橫向、標準與最大輔助文字尺寸的版面驗證中，主要標題、搜尋、新增、篩選、設定、貼上與歷史內容的水平裁切數為 0。
- **SC-005**: VoiceOver 掃描首頁、設定及新增流程時，所有互動控制的未命名數為 0、重複搜尋入口數為 0，且主要任務均可在不依賴視覺位置的情況下完成。
- **SC-006**: 所有 iOS 圖示控制的有效觸控區域量測均達 44 x 44 點以上，且相鄰控制不發生誤觸測試失敗。
- **SC-007**: 使用者能在 10 秒內從首頁完成手動新增文字項目，並在 5 秒內完成搜尋、篩選還原或進入設定的任一任務。
- **SC-008**: 連續 50 次快速明確貼上／取消／較新請求取代，並穿插前景與背景切換的壓力驗證中，閃退、重複保存及過期非同步結果寫入次數皆為 0。
- **SC-009**: iOS 變更完成後，macOS 既有首頁、持續自動捕捉、設定與原生列動作的自動回歸失敗數為 0。
- **SC-010**: 完整功能驗證中，所有剪貼簿匯入、儲存、搜尋、設定與歷史操作在沒有網路連線時均可完成，對外傳輸的剪貼簿衍生內容為 0。

## Assumptions

- 「複製後打開 iOS 就可以使用」表示打開後立即看到一個可用且明確的系統 Paste 動作；不表示 App 可在沒有使用者點擊時自動讀取其他 App 的剪貼簿。
- iOS 在背景、被暫停或未執行期間無法可靠持續監測剪貼簿，因此本功能不承諾保存期間被後續覆寫的中間項目。
- 文字與圖片的接受、驗證、去重、保存、排序與保留規則沿用既有產品行為。
- iPhone 是本次視覺驗收的主要裝置；iPad 必須保持可用且使用相同原生容器，但不新增獨立多欄資訊架構。
- 系統 paste control 與 App-specific Paste from Other Apps 設定的最終顯示與行為由目前 iOS 版本控制；基本功能不依賴使用者預先把程式化讀取設為 Allow。
- 這次不新增雲端同步、分析、廣告、外部處理或任何需要傳送剪貼簿內容的服務。

## Out of Scope

- 在 App 位於背景或未執行時擷取每一次剪貼簿變更。
- 在啟動、畫面出現或回前景時自動讀取其他 App 的 pasteboard value。
- 恢復已被其他複製操作覆寫的舊剪貼簿內容。
- 重新設計 macOS 主畫面、macOS Settings 或 macOS 持續監測流程。
- 變更 visionOS 的導覽與剪貼簿行為。
- 新增雲端同步、跨裝置剪貼簿歷史或遠端備份。
- 變更既有資料模型、內容去重定義或歷史排序規則。
