## Task Goal

- Establish a typed `HistoryLimit` preference with deterministic migration, fresh-install defaulting, and UserDefaults persistence.

## Skills Applied

- `speckit-implement` (scoped to T016 only)

## Root Cause

- `NextPasteApp` created `HistoryLimitPreference()` without supplying fresh-install detection, so production launch always followed the existing-install path.
- Missing or invalid persisted values were not repaired to a deterministic safe fallback.

## Implementation

- Kept `HistoryLimit` as the typed persistence model and tightened persistence loading so only supported presets and valid custom values are accepted.
- Repaired missing, corrupt, or semantically invalid stored values to the safe fallback `.unlimited`, while preserving valid existing values.
- Added deterministic launch-time install classification in `NextPasteApp` using existing app-defaults evidence plus persisted history evidence, then passed that result into `HistoryLimitPreference`.
- Extended unit coverage for fresh install default `500`, preserved persisted values, restart behavior, invalid persisted value fallback, and app-init detection rules.
